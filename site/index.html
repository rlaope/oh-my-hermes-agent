Hermes sees the installed skills first. Wrapper operators can inspect the same local pipeline contracts for research briefs, meeting prep, feedback triage, ops review, safe coding handoff, status, and release-readiness review. Operators can opt into omh setup --with-plugin for a local Hermes plugin bridge that only surfaces metadata-backed status.

Bootstrap when you need local operators.

Hermes skill tap is the primary entry point. The installer writes managed Hermes skills, registers the skill directory, and leaves runtime artifacts local and inspectable for wrappers and maintainers.

curl -fsSL https://raw.githubusercontent.com/rlaope/oh-my-hermes-agent/main/install.sh | sh omh setup omh doctor omh demo orchestration
OMH is deterministic local contract tooling. It does not patch Hermes core, run platform bots, call LLM APIs, or launch coding executors by itself.
