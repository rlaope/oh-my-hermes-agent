Situation playbooks for real wrapper work.

OMH can describe the whole local pipeline for common situations: research, deep interview, planning, safe coding handoff, status, and release-readiness review.

Research first

Keep source-backed questions with Hermes until evidence, confidence, and uncertainty are clear.

Plan before handoff

Turn broad goals into one-question interviews, clarified briefs, accepted plans, and only then prepared coding handoffs.

Status without overclaiming

Show handoff, execution, verification, review, CI, and merge-readiness as separate observed states.

omh playbook recommend "I want to safely add a feature to this repo" omh playbook inspect safe-feature-change

Natural chat in, evidence-backed next step out.

OMH keeps Hermes as the chat-facing orchestrator while external executors handle main coding work when a request needs code changes.

Chat-native routing

Wrappers can pass Discord, Slack, hosted, or generic messages into one deterministic interaction contract.

Retained cognition

Hermes keeps clarification, source-backed research, planning, and status narration first-class instead of forcing every request into coding.

Delegation-first coding

Coding-heavy requests become prepared Codex-like handoffs with scope, acceptance criteria, verification, and review expectations.

Install the preview channel.

The installer writes managed Hermes skills, registers the skill directory, and leaves runtime artifacts local and inspectable.

curl -fsSL https://raw.githubusercontent.com/rlaope/oh-my-hermes-agent/main/install.sh | sh omh doctor omh demo orchestration
OMH is deterministic local contract tooling. It does not patch Hermes core, run platform bots, call LLM APIs, or launch Codex by itself.
