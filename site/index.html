Natural chat in, evidence-backed next step out.

OMH keeps Hermes as the chat-facing orchestrator while external executors handle main coding work when a request needs code changes.

Chat-native routing

Wrappers can pass Discord, Slack, hosted, or generic messages into one deterministic interaction contract.

Retained cognition

Hermes keeps clarification, source-backed research, planning, and status narration first-class instead of forcing every request into coding.

Delegation-first coding

Coding-heavy requests become prepared Codex-like handoffs with scope, acceptance criteria, verification, and review expectations.

Contracts that wrappers can render without guessing.

OMH exposes machine-readable envelopes for routing, plans, status cards, harness quality, and harness progress. Prepared handoffs never count as observed implementation, review, CI, or merge evidence.

OMH is deterministic local contract tooling. It does not patch Hermes core, run platform bots, call LLM APIs, or launch Codex by itself.
