Handoff lifecycle

"Turn this issue into a PR" should become scope, non-goals, acceptance criteria, verification commands, review expectations, and executor choice before anyone claims code changed.

request plan accepted executor choice dispatch observed evidence

Evidence boundary

  • Prepared handoff is not implementation evidence.
  • Executor dispatch must be observed separately.
  • Review, CI, and merge readiness need their own records.
  • Prompt-only executors stay prompt-only until a tested lifecycle exists.
