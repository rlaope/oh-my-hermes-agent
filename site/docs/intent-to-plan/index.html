Flagship lane

Intent to plan

Use this lane when a user gives Hermes a fuzzy product, technical, or goal-shaped request. Hermes should not jump straight to coding. It should clarify, plan, lock acceptance criteria, and then create a durable goal or handoff only when the shape is ready.

Plain request

"Make onboarding feel smoother" is not yet a coding task. It needs intent, non-goals, user value, acceptance criteria, and a plan that can survive review.

Use these skills

  • deep-interview asks the smallest blocking question first.
  • ralplan turns the clarified intent into a reviewed plan.
  • ultragoal records durable progress and completion gates.
  • loop takes over when the goal should keep cycling.
